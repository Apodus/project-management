# Campaign — Per-project auto-implement settings (web-toggleable, default off)

**Goal:** Move the responder auto-implement enablement from daemon-wide env vars to a **per-project, DB-backed, web-toggleable** setting (default OFF) so the operator can enable it for game_one from the web UI while new clients stay off automatically — the per-project granularity env vars can't give (env is per-daemon, governing all watched projects).
**Branch:** `campaign-responder-per-project-settings` (off main). **Do NOT push** until the user approves the landing.

## Where we are (the accurate current state)
Today ALL auto-implement config lives in the responder daemon's env/CLI (`packages/responder-ref/src/config.ts`): `PM_AUTO_IMPLEMENT_ENABLED` (default false), `PM_AUTO_IMPLEMENT_MODE` (default on), budget/allowlist/verify/git-url. The PM server/web has NO per-project control — the A5 dashboard SHOWS auto-implement activity but doesn't CONTROL it. (The CLAUDE.md A1+A2 paragraph's claim that this config "lives in settings.integrator (responder block)" is **inaccurate** — it shipped as env vars; P4 fixes that line.)

## The established pattern to reuse (read first)
`packages/shared/src/schemas/project.ts` — `projectSettingsSchema` has per-feature sub-blocks (`integrator`, `webhooks`, `ai_autonomy`, `git`, …), each `.optional()`, **written PARTIALLY** (each web settings page read-merge-writes ONE sub-block) and **read TOLERANTLY** (consumers fall back to per-field defaults). `integratorSettingsSchema` (project.ts:217) is the model. The Zod-4 route mirror lives in `packages/server/src/routes/projects.ts` (the established Zod-3-canonical / Zod-4-route-mirror split — keep them in lockstep). The web has an Integrator settings page (`/projects/{id}/settings/integrator`) + a per-project settings nav; mirror it.

## The authoritative composition design (resolve in P2 — stated here so every sub-agent aligns)
- **Per-project source of truth:** `settings.autoImplement.enabled` (default **false**) is the per-project switch; `settings.autoImplement.mode` (default **"shadow"** — the safe observe-first rung for a DB opt-in, deliberately distinct from the env default "on" which exists for A1-A4 byte-identity).
- **The responder composes per-project, per tick:** for each watched project it fetches `project.settings.autoImplement` from the API (it already polls per project) and resolves:
  - `effectiveEnabled(project) = dbSettings.enabled === true AND envMasterNotExplicitlyOff`. The env `PM_AUTO_IMPLEMENT_ENABLED` becomes a daemon-wide **master kill-switch**: explicitly false ⇒ force OFF for ALL projects (belt-and-suspenders); **true OR UNSET ⇒ defer to the per-project DB**. Semantic shift: env-unset now defers to the DB instead of forcing off — but the default stays OFF everywhere (DB defaults off), so a responder shipping with the env unset is **byte-identical** (still off until the operator flips a project's DB toggle).
  - `effectiveMode(project) = dbSettings.mode ?? env PM_AUTO_IMPLEMENT_MODE ?? "shadow"`.
- **Deployment knobs STAY env (daemon-wide):** `PM_RESPONDER_GIT_REPO_URL` (clone source), the budget caps, `allowedPaths`, `verifyCmd`. v1 exposes only `enabled` + `mode` per-project (the highest-value rollout knobs — exactly what the user wants); budget/allowlist could move to the DB in a later pass.
- **Git-url is fail-safe, NOT a startup crash:** drop the "repoUrl required iff enabled" STARTUP ConfigError (the daemon can't know per-project enablement at boot). Instead, if a project resolves effective-enabled but no git url is configured, the existing worktree-prep failure path escalates-to-human / logs (fail-safe — no implement, no crash). Document: set `PM_RESPONDER_GIT_REPO_URL` on any responder that will have per-project auto-implement enabled.

## Engineering values
No investment ceiling; reuse the integrator-settings pattern (schema + route mirror + web page + nav) wholesale; additive — every existing project with no `autoImplement` block is OFF + byte-identical; the whole A1-A5 arc stays byte-identical (only the ENABLEMENT SOURCE moves from env-only to DB-composed-with-env). Ships safe-by-default (default off, default mode shadow).
